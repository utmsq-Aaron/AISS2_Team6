Chapter: 3_state_of_the_art, Section: AI-Assisted Sports Coaching (line 86)
Claim: [CITE] proposed SMART-ALEC, a multi-agent health coaching system with six internal modules per user agent (planner, scheduler, learner, context analyser, executor, communicator) for physical activity promotion. They implemented two prototypes: BeSmart (SMS-based, beta-tested with 250 agents) and a JIT intervention system using Apple Watch with nightly-updated decision trees. While predating MCP and A2A and using ad-hoc coordination (GPGP-style commitment exchange), their design philosophy---modular specialists with a central coordinator, transferable learned policies between similar users---validates the multi-agent approach.
Source quote: "we propose a novel generic multi-agent architecture, SMART-ALEC (System to Monitor, Adapt, and Refine Target goals by Agent-based Learning in Everyday Contexts)" (p. 2, Section 1)
Source quote: "Each agent has two major modules, analyser and communicator, running concurrently on different threads." (p. 3, Section 3) [The Analyser contains: (Re)Planner, Scheduler, Learner, Context Analyzer, Executor/Monitor -- visible in Figure 1, p. 10]
Source quote: "Planner. SMART-ALEC divides top-level goal achievement into three levels: planning, scheduling and learning." (p. 3, Section 3)
Source quote: "Scheduler. The purpose of scheduler is to determine which actions can be executed now" (p. 4, Section 3)
Source quote: "Learner. The purpose of learner is to construct policies in domains where how to achieve goals is not explicitly given beforehand." (p. 4, Section 3)
Source quote: "Context Analyzer. Context analyzer is used for constructing user state in real time." (p. 4, Section 3)
Source quote: "Executor/Monitor. [...] The executor module not only implements the current complied policy but also acts as a feedback provider for the learner module." (p. 4, Section 3)
Source quote: "Communicator. The communicator module is responsible for transferability and coordination features of SMART-ALEC architecture. The communicator can transfer learned policies of users to other similar users." (p. 4, Section 3)
